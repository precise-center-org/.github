# Contributing to the Precise Center

Thank you for your interest in contributing to the Precise Center for Precision Rehabilitation! We welcome contributions from Center members, collaborators, and the broader rehabilitation community. We aim to make precision rehabilitation accessible to all by sharing high-quality, well-documented, and generalizable resources that can be used beyond their immediate use case.
  
### Before You Share a Resource

1️⃣ **Make it generalizable** - Ensure your resource can be adapted for use beyond your specific project  
2️⃣ **Document thoroughly** - Include clear instructions, examples, and context  
3️⃣ **Test it** - Verify that others can understand and use your resource  
4️⃣ **Obtain permissions** - Secure all necessary institutional approvals and co-author agreements  
5️⃣ **Contact us** - Reach out to the Resource Core team before uploading

Read on for more details!

## 🚀 How to Contribute?
<details>
  <summary>Click to expand</summary>
  
### Step 1: Contact the Resource Core Team

Before creating a new repository, reach out to:
- **Email:** PreciseCenter@kennedykrieger.org
- **Subject line:** "Resource Contribution: [Brief Description]"

Include:
- Brief description of your resource
- Who it's for (clinicians, researchers, both)
- What problem it solves
- Current state (ready to share, needs refinement, etc.)
- Confirmation that you have necessary permissions to share

### Step 2: Prepare Your Resource

Follow our **Resource Preparation Checklist**:

☐ Create a clear, descriptive name for your resource<br>
☐ Write a comprehensive README.md file<br>
☐ Add example data or test cases (if applicable)<br>
☐ Include license information (we recommend open-source licenses)<br>
☐ Document all dependencies and requirements<br>
☐ Test with someone unfamiliar with your project<br>
☐ Obtain all necessary institutional approvals and co-author permissions<br>

### Step 3: Create or Upload to Repository

#### Option A: Create a New Repository (Recommended)
1. Create repository under the `precise-center-org` organization
2. Use a clear, descriptive name (e.g., `ai-gait-analysis`, `clinical-battery-toolkit`)
3. Make it public (unless there's a specific reason for private)
4. Add appropriate topics/tags for discoverability

#### Option B: Add to Existing Repository
If your resource fits within an existing repository, coordinate with the repository maintainer

### Step 4: Link from Main Page

Update the main organization README to include your resource in the appropriate category

</details>

## 🔄 Transferring Existing Repositories
<details>
  <summary>Click to expand</summary>
  
If you've already created a resource repository on your personal GitHub account and want to contribute it to the Precise Center:

### Transfer Your Repository

1. Go to your repository's **Settings**
2. Scroll to **Danger Zone** → **Transfer ownership**
3. Enter organization name: `precise-center-org`
4. Confirm transfer

After transfer:
- You'll be added as a maintainer of the repository
- Your contribution history is preserved
- The repository now lives under the Precise Center organization
- You can continue to develop and maintain it

**Questions?** Contact the Resource Core team before transferring.

</details>

## 📋 Contribution Guidelines
<details>
  <summary>Click to expand</summary>

### Quality Standards

All contributions should meet these criteria:

#### 1. **Documentation**
- Clear README file explaining what the resource does and how to use it
- Installation/setup instructions (if applicable)
- Examples or tutorials demonstrating usage
- Citations for any methods or tools used

#### 2. **Code Quality** (for software/analysis code)
- Well-commented code explaining key sections
- Clear variable and function names
- Dependencies clearly listed
- Example data or test cases included

#### 3. **Accessibility**
- Designed to be usable with minimal technical expertise (when possible)
- Clear language avoiding unnecessary jargon
- Step-by-step instructions
- Troubleshooting guidance

#### 4. **Reproducibility**
- Version numbers for all software/packages used
- Random seeds set (for analyses involving randomization)
- Data formats clearly specified

</details>

## ❓ Frequently Asked Questions
<details>
  <summary>Click to expand</summary>
  
### General Questions

**Q: I'm not very technical. Can I still contribute?**  
A: Absolutely! We welcome all types of resources including clinical protocols, questionnaires, and other educational materials. The Resource Core team can help you determine the best format and method in which to upload your contribution.

**Q: I'm new to GitHub. Where do I begin?**
A: Check out [GitHub's Getting Started Guide](https://docs.github.com/en/get-started). Need help with Git? See [Git Basics](https://git-scm.com/book/en/v2/Getting-Started-Git-Basics)

**Q: Do I need to be a Precise Center member to contribute?**  
A: Center members and collaborators are encouraged to contribute. External contributors are welcome but should contact us first at PreciseCenter@kennedykrieger.org to discuss their proposed contribution.

**Q: What if my resource isn't "finished" yet?**  
A: That's okay! Contact us to discuss. We can help you refine it, or you can share it as a "work in progress" with appropriate labeling.

### Repository Questions

**Q: I already have a repository on my personal GitHub. How do I add it to the organization?**  
A: You can transfer ownership of your repository to `precise-center-org`. See the "Transferring Existing Repositories" section above. You'll retain access and your contribution history is preserved.

**Q: Will I lose credit for my work if I transfer my repository?**  
A: No! Your contribution history is fully preserved. Your commits will still show your name, and it will appear in your GitHub contribution graph. Plus, we'll credit you in the repository and any publications.

**Q: Can I keep working on a repository after I transfer it?**  
A: Yes! After transfer, you'll be added as a maintainer/collaborator with appropriate permissions. You can continue developing and maintaining your resource.

**Q: What if I want to keep a copy on my personal account too?**  
A: You can fork the organization's repository back to your personal account after transfer, but we recommend just working directly with the organization repository to avoid duplication.

**Q: Should I create a new repository or add to an existing one?**  
A: Contact the Resource Core team to discuss. If your resource fits naturally within an existing repository (e.g., adding a new analysis script to a code library), we'll help integrate it. If it's standalone, create a new repository.

### Documentation Questions

**Q: How detailed should my documentation be?**  
A: Think about someone unfamiliar with your project trying to use it. Include:
- What it does and why it's useful
- Requirements and dependencies
- Step-by-step instructions
- At least one example
- Troubleshooting tips for common issues

**Q: I'm not good at writing documentation. Can you help?**  
A: Yes! The Resource Core team can help you improve your documentation. Just share what you have and we'll work with you to enhance it.

**Q: Do I need to include example data?**  
A: Whenever possible, yes. Example or test data helps users understand how to use your resource. Make sure you have appropriate permissions if using real data, or create synthetic examples.

### Technical Questions

**Q: What license should I use?**  
A: It depends on your resource type:

- **For code/software:** MIT or Apache 2.0 (permissive open-source licenses)
- **For documentation, protocols, or educational materials:** Creative Commons (CC BY 4.0 recommended)
- **For datasets:** CC0 or CC BY 4.0

The Resource Core team can help you choose the most appropriate license for your specific resource.

**Q: What programming languages are acceptable?**  
A: Any language! Common ones in rehabilitation research include Python, R, MATLAB, and JavaScript. Just make sure to document dependencies clearly.

**Q: My code uses proprietary software (like MATLAB). Can I still share it?**  
A: Yes, but note the requirements clearly in your documentation. Consider providing alternatives when possible (e.g., Python or R versions).

**Q: How do I handle updates and versioning?**  
A: Use GitHub releases to mark significant versions (e.g., v1.0, v2.0). Document changes in a CHANGELOG.md file. For minor updates, just commit and push changes with clear commit messages.

### Collaboration Questions

**Q: Can multiple people contribute to the same repository?**  
A: Yes! We can add multiple collaborators to any repository. Coordinate with co-contributors about who manages what.

**Q: What if someone wants to build on my resource?**  
A: That's great! That's the whole point of open science. They can fork your repository, make improvements, and submit pull requests. You can review and merge their contributions.

**Q: How do I get help with my contribution?**  
A: Contact the Resource Core team at PreciseCenter@kennedykrieger.org. We're here to help with:
- Technical questions
- Documentation
- Best practices
- GitHub workflows
- Making your resource more generalizable

### Usage and Impact Questions

**Q: How will I know if people are using my resource?**  
A: GitHub provides metrics like:
- Stars (people bookmarking your repository)
- Forks (people copying it to work on)
- Clones (people downloading it)
- Traffic analytics

**Q: Can I include my GitHub contributions in my CV/publications?**  
A: Absolutely! Open-source contributions are increasingly valued in academia. Include:
- Repository URL
- Your role (creator, maintainer, contributor)
- Impact metrics (stars, forks, citations)

**Q: How should my resource be cited?**  
A: Include a citation section in your README with this format:
```markdown
## Citation
If you use this resource, please cite:  
[Your Name]. (Year). [Resource Name]. The Center for Precision Rehabilitation Across the Lifespan (NIH P50HD118624). GitHub repository: https://github.com/precise-center-org/[repo-name]
```
Consider getting a DOI through Zenodo for formal citation tracking.

**Q: What if my resource is used in a publication?**  
A: We encourage users to cite resources they use. You can track citations through Google Scholar (if you have a DOI) or by searching for mentions of your repository.
